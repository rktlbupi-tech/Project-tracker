# Guide: Deploying Your Node.js Backend to AWS EC2

This guide will help you get your Express backend live on your Ubuntu EC2 instance. Since your connection was reset, the first step is to reconnect.

---

## 1. Reconnect to Your EC2 Instance
First, ensure you are back in your terminal on your Mac and reconnect to your EC2 instance.
```bash
ssh -i "your-key-pair.pem" ubuntu@your-ec2-public-ip
```

## 2. Install Node.js (via NVM)
Using NVM (Node Version Manager) is the best way to manage Node versions on Ubuntu.
```bash
# Install NVM
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Load NVM
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

# Install Node (LTS version)
nvm install --lts
nvm use --lts
```

## 3. Clone Your Repository
Clone your project onto the EC2 instance.
```bash
# If you haven't already
git clone https://github.com/your-username/Project-tracker.git
cd Project-tracker/backend
```

## 4. Environment Variables
Your project depends on `.env`. You need to create this on the server manually because `.env` files are usually not pushed to Git.
```bash
nano .env
```
*Paste your production environment variables here and press `Ctrl+O`, `Enter`, `Ctrl+X` to save and exit.*

## 5. Install Dependencies and Test
```bash
npm install
# Test the start (stop it with Ctrl+C after verifying it works)
node server.js
```

## 6. Use PM2 (Process Manager)
To keep your backend running even after you close your terminal:
```bash
# Install PM2 globally
npm install pm2 -g

# Start your server
pm2 start server.js --name "project-backend"

# Ensure PM2 restarts on server reboot
pm2 startup
# (Copy and paste the command generated by the output of above)
pm2 save
```

## 7. Set Up Nginx Reverse Proxy (Optional but Recommended)
Instead of accessing via `http://your-ip:5000`, Nginx allows you to use standard port 80/443.
```bash
sudo apt update
sudo apt install nginx -y

# Configure Nginx
sudo nano /etc/nginx/sites-available/default
```
Replace the `location /` block with:
```nginx
location / {
    proxy_pass http://localhost:5000; # Use the port your server runs on
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_cache_bypass $http_upgrade;
}
```
Save and restart Nginx:
```bash
sudo systemctl restart nginx
```

## 8. AWS Security Groups (CRITICAL)
Go to your **AWS Console** > **EC2** > **Security Groups** and add an **Inbound Rule**:
- **Type**: HTTP (Port 80)
- **Source**: 0.0.0.0/0
- (If not using Nginx) **Type**: Custom TCP (Port 5000)
- **Source**: 0.0.0.0/0

---

> [!TIP]
> If you encounter "Connection reset by peer" again, it might be due to your network or SSH timeout. You can add `ServerAliveInterval 60` to your local `~/.ssh/config` to keep the connection alive.
